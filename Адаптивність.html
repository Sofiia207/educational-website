Контрольні точки

Контрольні точки - це ширина, що настроюється, яка визначає поведінку адаптивного макета на різних пристроях або розмірах області перегляду в Bootstrap.

Основні концепції

  • Контрольні точки – це будівельні блоки адаптивного дизайну. Використовуйте їх, щоб контролювати, коли макет може бути адаптований для певного вікна перегляду або розміру пристрою.
  • Використовуйте медіа-запити для створення Вашого CSS за контрольною точкою. Медіа-запити - це функція CSS, яка дозволяє умовно застосовувати стилі на основі набору параметрів браузера та операційної системи. Найчастіше використовують min-width у медіа-запитах.
  • Насамперед мобільні, мета – адаптивний дизайн. CSS Bootstrap має на меті застосування мінімальної кількості стилів, щоб макет працював з мінімальною контрольною точкою, а потім на шари стилів, щоб налаштувати цей дизайн для більших пристроїв. Це оптимізує Ваш CSS, скорочує час рендерингу та забезпечує відмінний досвід для Ваших відвідувачів.

Доступні контрольні точки

Bootstrap включає шість контрольних точок за умовчанням, іноді званих grid tiers, для швидкої побудови.

Контрольна точка Інфікс класу Габаритні розміри
Дуже маленький None < 576px
Маленький sm ≥ 576px
Середній md ≥ 768px
Великий lg ≥ 992px
Дуже великий xl ≥ 1200px
Огромний xxl ≥ 1400px

Кожна контрольна точка була обрана для зручного розміщення контейнерів, ширина яких кратна 12. Контрольні точки також є підмножиною загальних розмірів пристроїв і розмірів області перегляду - вони спеціально не націлені на кожен варіант використання або пристрій. Навпаки, ці лінійки забезпечують міцну та стабільну основу практично для будь-якого пристрою.
Ці контрольні точки налаштовуються через Sass – Ви знайдете їх на карті Sass у таблиці стилів _variables.scss.


    $grid-breakpoints: (
        xs: 0,
        sm: 576px,
        md: 768px,
        lg: 992px,
        xl: 1200px,
        xxl: 1400px
      );

Медіа-запити

Оскільки Bootstrap в першу чергу розроблено для мобільних пристроїв, ми використовуємо кілька медіа-запитів для створення розумних контрольних точок для наших макетів та інтерфейсів. Ці контрольні точки в основному ґрунтуються на мінімальній ширині області перегляду і дозволяють масштабувати елементи в міру зміни області перегляду.

Мінімальна ширина

Bootstrap в основному використовує наступні діапазони медіа-запитів – або контрольні точки – у наших вихідних файлах Sass для розмітки, системи сіток та компонентів.


    // Исходные миксины

    // Нет необходимости в медиа-запросе для контрольной точки xs, так как это эффективно `@media (min-width: 0) { ... }`
    @include media-breakpoint-up(sm) { ... }
    @include media-breakpoint-up(md) { ... }
    @include media-breakpoint-up(lg) { ... }
    @include media-breakpoint-up(xl) { ... }
    @include media-breakpoint-up(xxl) { ... }
    
    // Применение
    
    // Пример: Скрыть начало с `min-width: 0`, а затем показать в контрольной точке `sm`
    .custom-class {
      display: none;
    }
    @include media-breakpoint-up(sm) {
      .custom-class {
        display: block;
      }
    }

Ці міксини Sass перетворюються на наш скомпільований CSS з використанням значень, оголошених у наших змінних Sass. Наприклад:


    // Устройства X-Small (портретные телефоны, менее 576 пикселей)
    // Нет медиа-запроса для `xs`, так как это значение по умолчанию в Bootstrap
    
    // Устройства Small (телефоны с горизонтальной ориентацией, 576 пикселей и выше)
    @media (min-width: 576px) { ... }
    
    // Устройства Medium (планшеты, 768 пикселей и выше)
    @media (min-width: 768px) { ... }
    
    // Устройства Large (настольные компьютеры, 992 пикселей и выше)
    @media (min-width: 992px) { ... }
    
    // Устройства X-Large (большие настольные компьютеры, 1200 пикселей и выше)
    @media (min-width: 1200px) { ... }
    
    // Устройства XX-Large (большие настольные компьютеры, 1400 пикселей и выше)
    @media (min-width: 1400px) { ... }

Максимальна ширина

Іноді використовують медіа-запити в іншому напрямку (заданий розмір екрана або менше):


    // `xs` возвращает только набор правил, но не медиа-запрос
    // ... { ... }
    
    // `sm` применяется к устройствам x-small (портретные телефоны, менее 576 пикселей)
    @media (max-width: 575.98px) { ... }
    
    // `md` применяется к небольшим устройствам medium (телефоны с горизонтальной ориентацией, менее 768 пикселей)
    @media (max-width: 767.98px) { ... }
    
    // `lg` применяется к устройствам среднего размера large (планшеты, менее 992 пикселей)
    @media (max-width: 991.98px) { ... }
    
    // `xl` применяется к большим устройствам (настольные компьютеры, менее 1200 пикселей)
    @media (max-width: 1199.98px) { ... }
    
    // `xxl` применяется к устройствам x-large (большие рабочие столы, менее 1400 пикселей)
    @media (max-width: 1399.98px) { ... }

Поодинока контрольна точка

Існують також медіа-запити та міксини для націлювання на один сегмент розмірів екрану з використанням мінімальної та максимальної ширини контрольної точки.


    @include media-breakpoint-only(xs) { ... }
@include media-breakpoint-only(sm) { ... }
@include media-breakpoint-only(md) { ... }
@include media-breakpoint-only(lg) { ... }
@include media-breakpoint-only(xl) { ... }
@include media-breakpoint-only(xxl) { ... }

Наприклад, @include media-breakpoint-only(md) { ... } призведе до:


    @media (min-width: 768px) and (max-width: 991.98px) { ... }

Між контрольними точками

Так само медіа-запити можуть охоплювати кілька значень ширини контрольної точки:


    @include media-breakpoint-between(md, xl) { ... }

Що приводить до:


    // Пример
// Применятся стили, начиная со средних и заканчивая очень большими.
@media (min-width: 768px) and (max-width: 1199.98px) { ... }
