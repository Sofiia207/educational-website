Розпочати роботу з Bootstrap

Швидкий старт:

Створіть новий файл index.html у корені вашого проекту. Увімкніть також тег <meta name="viewport"> для правильної чуйної поведінки на мобільних пристроях.


                <!doctype html>
                <html lang="uk">
                  <head>
                    <meta charset="utf-8">
                    <meta name="viewport" content="width=device-width, initial-scale=1">
                    <title>Демо Bootstrap</title>
                  </head>
                  <body>
                    <h1>Привіт, світ!</h1>
                  </body>
                </html>

Увімкніть CSS та JS Bootstrap:

Помістіть тег <link> у <head> для нашого CSS і тег <script> для нашого пакету JavaScript (включаючи Popper для позиціонування розкривних списків, спливаючі повідомлення та підказки) перед закриваючим </body>.


                        <!doctype html>
                <html lang="ru">
                <head>
                    <meta charset="utf-8">
                    <meta name="viewport" content="width=device-width, initial-scale=1">
                    <title>Демо Bootstrap</title>
                    <link href="https://cdn.jsdelivr.net/npm/bootstrap@5.3.3/dist/css/bootstrap.min.css" rel="stylesheet" integrity="sha384-QWTKZyjpPEjISv5WaRU9OFeRpok6YctnYmDr5pNlyT2bRjXh0JMhjY6hW+ALEwIH" crossorigin="anonymous">
                </head>
                <body>
                    <h1>Привіт, світ!</h1>
                    <script src="https://cdn.jsdelivr.net/npm/bootstrap@5.3.3/dist/js/bootstrap.bundle.min.js" integrity="sha384-YvpcrYf0tY3lHB60NNkmXc5s9fDVZLESaAA55NDzOxhy9GkcIdslK1eN7N6jIeHz" crossorigin="anonymous"></script>
                </body>
                </html>

Ви також можете включити Popper та JS окремо. Якщо ви не плануєте використовувати розкривні списки, спливаючі вікна або підказки, заощаджуйте кілька кілобайт, не включаючи Popper.



            <script src="https://cdn.jsdelivr.net/npm/@popperjs/core@2.11.8/dist/umd/popper.min.js" integrity="sha384-I7E8VVD/ismYTF4hNIPjVp/Zjvgyol6VFvRkX/vR+Vc4jQkC+hVqc2pM8ODewa9r" crossorigin="anonymous">
            <script src="https://cdn.jsdelivr.net/npm/bootstrap@5.3.3/dist/js/bootstrap.min.js" integrity="sha384-0pUGZvbkm6XF6gxjEnlmuGrJXVbNuzT9qBBavbLwCsOGabYfZo0T0to5eqruptLy" crossorigin="anonymous">

Привіт світ!

Відкрийте сторінку у вибраному браузері, щоб побачити сторінку з початковим завантаженням. Тепер ви можете почати збирання за допомогою Bootstrap, створивши свій власний макет, додавши десятки компонентів використовуючи наші підказки на науступних сторінках.

Важливі глобальні змінні

HTML5 doctype:

Bootstrap вимагає використання типу документу HTML5. Без нього ви побачите химерний та незавершений стиль.


            <!doctype html>
            <html> lang="uk">
              ...
              </html>

Viewport meta

Bootstrap розроблений спочатку для мобільних пристроїв, стратегія, в якій спочатку оптимізуємо код для мобільних пристроїв, а потім масштабуємо компоненти за необхідності, використовуючи медіа-запити CSS. Щоб забезпечити правильний рендеринг та сенсорне масштабування для всіх пристроїв, додайте метатег адаптивного вікна перегляду до <head>.


                <meta name="viewport" content="width=device-width, initial-scale=1">

Box-sizing

Для більш простого визначення розміру CSS ми перемикаємо глобальне значення box-sizing з content-box на border-box. Це гарантує, що padding не вплине на остаточну обчислену ширину елемента, але може викликати проблеми з деякими сторонніми програмами, такими як Google Maps та Google Custom Search Engine.
У поодиноких випадках вам потрібно перевизначити його, використовуйте щось на зразок такого:


                .selector-for-some-widget {
                box-sizing: content-box;
                }

У наведеному вище фрагменті вкладені елементи, включаючи згенерований контент за допомогою :: before та :: after, будуть успадковувати зазначений box-sizing для цього .selector-for-some-widget.

Reboot

Для покращення крос-браузерного рендерингу використовується Reboot для виправлення невідповідностей між браузерами та пристроями, забезпечуючи при цьому дещо самостійніше скидання загальних елементів HTML.

Браузери та пристрої

Підтримувані браузери

Bootstrap підтримує останні стабільні випуски всіх основних браузерів та платформ.
Альтернативні браузери, які використовують останню версію WebKit, Blink або Gecko, безпосередньо або через API веб-подання платформи, явно не підтримуються. Однак Bootstrap повинен (у більшості випадків) правильно відображатися та працювати у цих браузерах. Більш конкретну інформацію про підтримку наведено нижче.


        <meta name="viewport" content="width=device-width, initial-scale=1">
        # https://github.com/browserslist/browserslist#readme

        >= 0.5%
        last 2 major versions
        not dead
        Chrome >= 60
        Firefox >= 60
        Firefox ESR
        iOS >= 12
        Safari <= 12
        not Explorer <= 11

Настільні браузери

Так само підтримуються останні версії більшості настільних браузерів.

Для Firefox, крім останньої нормальної стабільної версії, також підтримується останню версію Firefox Extended Support Release (ESR) version of Firefox.
Неофіційно Bootstrap має виглядати і поводитися досить добре у Chromium та Chrome для Linux та Firefox для Linux, хоча офіційно вони не підтримуються.

Internet Explorer

Internet Explorer не підтримується. Якщо потрібна підтримка Internet Explorer, використовуйте Bootstrap v4.

Модальні вікна та списки, що випадають на мобільних пристроях

Переповнення та прокручування

Підтримка overflow: hidden; в елементі <body> дуже обмежена в iOS та Android. З цією метою, коли ви прокручуєте верхню або нижню частину модального вікна в браузері будь-якого з цих пристроїв, вміст починає прокручуватися.

Текстові поля iOS та прокручування

Починаючи з iOS 9.2, поки модальне вікно відкрито, якщо початковий дотик жесту прокручування знаходиться в межах текстового <input> або <textarea>, вміст <body> під модальним вікном прокручуватиметься замість самого модального вікна.

Випадаючі меню навігаційної панелі

Елемент .dropdown-backdrop не використовується в навігаційній панелі iOS через складність z-індексу. Таким чином, щоб закрити списки, що розкриваються на панелі навігації, Ви повинні безпосередньо клацнути елемент розкривного списку (або будь-який інший елемент, який викличе подію клацання в iOS).

Масштабування у браузері

При масштабуванні сторінки неминуче з'являються артефакти рендерингу в деяких компонентах як в Bootstrap, так і в іншій частині Інтернету. Залежно від проблеми ми можемо її виправити (спочатку виконайте пошук, а потім у разі необхідності відкрийте проблему).

Валідатори

Щоб забезпечити максимальну зручність роботи старих браузерів та браузерів з помилками, Bootstrap використовує хакі браузера CSS у кількох місцях для націлювання спеціального CSS на певні версії браузера, щоб уникнути помилок у самі браузери. Ці хакі зі зрозумілих причин змушують валідаторів CSS скаржитися на їхню недійсність. У деяких випадках також використовуються передові функції CSS, які ще не повністю стандартизовані, але використовуються виключно для прогресивного покращення.
Ці попередження про перевірку не мають значення на практиці, оскільки не хакерська частина CSS дійсно повністю перевіряє, а хакерські частини не заважають правильному функціонуванню не хакерської частини.

Sass

Використовуйте вихідні файли Sass, щоб скористатися перевагами змінних, карт, домішок та функцій, які допоможуть Вам швидше створювати та налаштовувати свій проект.

Файлова структура

Уникайте зміни файлів ядра Bootstrap. Для Sass це означає створення власної таблиці стилів, яка імпортує Bootstrap, щоб ви могли змінювати та розширювати її. Припускаючи, що ви використовуєте менеджер пакетів, такий як npm, у вас буде файлова структура, яка виглядає так:


    your-project/
├── scss/
│   └── custom.scss
└── node_modules/
│   └── bootstrap/
│       ├── js/
│       └── scss/
└── index.html

Якщо ви завантажили вихідні файли і не використовуєте менеджер пакетів, ви можете створити вручну щось схоже на цю структуру, зберігаючи вихідні файли Bootstrap окремо від ваших власних.


    your-project/
    ├── scss/
    │   └── custom.scss
    ├── bootstrap/
    │   ├── js/
    │   └── scss/
    └── index.html

Імпорт

У свій custom.scss Ви імпортуєте вихідні файли Sass для Bootstrap. У Вас є два варіанти: увімкнути весь Bootstrap або вибрати потрібні Вам частини. Ми заохочуємо останнє, але майте на увазі, що між компонентами є деякі вимоги та залежності. Вам також необхідно буде включити JavaScript для плагінів.


    // Custom.scss
// Вариант А: Включити увесь Bootstrap

// Include any default variable overrides here (though functions won't be available)

@import "../node_modules/bootstrap/scss/bootstrap";

// Then add additional custom code here


    // Custom.scss
// Вариант Б. Включити частини Bootstrap

// 1. Спочатку ввімкініть функції (щоб ви могли керувати кольорами, SVG, обчисленням і т.д.)
@import "../node_modules/bootstrap/scss/functions";

// 2. Включіть сюди будь-які предвизначення змінних за замовчуванням

// 3. Ввімкінть інщі потрібні таблиці стилей Bootstrap (включаючи будь-які таблиці стилей кольоровго режиму)
@import "../node_modules/bootstrap/scss/variables";
@import "../node_modules/bootstrap/scss/variables-dark";

// 4. Ввімкніть сюди будь-які превизначення карт за замовчуванням

// 5. Include remainder of required parts
@import "../node_modules/bootstrap/scss/maps";
@import "../node_modules/bootstrap/scss/mixins";
@import "../node_modules/bootstrap/scss/root";

// 6. За необхідності ввімкіть будь-які інщі частини
@import "../node_modules/bootstrap/scss/utilities";
@import "../node_modules/bootstrap/scss/reboot";
@import "../node_modules/bootstrap/scss/type";
@import "../node_modules/bootstrap/scss/images";
@import "../node_modules/bootstrap/scss/containers";
@import "../node_modules/bootstrap/scss/grid";
@import "../node_modules/bootstrap/scss/helpers";

// 7. За бажанням додайте API утиліт в останню чергу, щоб згенерувати класи на основі мапи Sass в `_utilities.scss`
@import "../node_modules/bootstrap/scss/utilities/api";

// 8. Додайте сюди додатковий користувацький код

З цією настройкою, Ви можете розпочати зміну будь-яких змінних і карт Sass у Вашому custom.scss. Ви також можете почати додавати частини Bootstrap у розділі // Optional у міру необхідності. Ми пропонуємо використовувати повний стек імпорту з нашого файлу bootstrap.scss як відправна точка.

Компіляція

Щоб використовувати ваш код Sass як CSS в браузері, вам потрібен компілятор Sass. Sass поставляється у вигляді пакета CLI, але ви також можете скомпілювати його за допомогою інших інструментів збирання, таких як Gulp або Webpack, або за допомогою програми з графічним інтерфейсом. Деякі IDE також мають вбудовані компілятори Sass або розширення, що завантажуються.
Радимо використовувати CLI для компіляції Sass, але ви можете використовувати будь-який метод, який ви віддаєте перевагу. З командного рядка виконайте таке:


    # Install Sass globally
npm install -g sass

# Watch your custom Sass for changes and compile it to CSS
sass --watch ./scss/custom.scss ./css/custom.css

Увімкнення

Як тільки ваш CSS скомпілюваний, ви можете включити його до файлів HTML. Всередині вашого index.html ви захочете увімкнути скомпільований файл CSS. Обов'язково оновіть шлях до скомпільованого файлу CSS, якщо ви його змінили.


    <!doctype html>
<html lang="en">
  <head>
    <meta charset="utf-8">
    <meta name="viewport" content="width=device-width, initial-scale=1">
    <title>Custom Bootstrap</title>
    <link href="/css/custom.css" rel="stylesheet">
  </head>
  <body>
    <h1>Hello, world!</h1>
  </body>
</html>

Значення змінних за замовчуванням

Кожна змінна Sass у Bootstrap включає прапор !default, що дозволяє перевизначити значення змінної за замовчуванням у Вашому власному Sass без зміни вихідного коду Bootstrap. Скопіюйте та вставте змінні при необхідності, змініть їх значення та видаліть прапор !default. Якщо змінна вже була призначена, вона не буде повторно призначена за замовчуванням у Bootstrap.
Ви знайдете повний список змінних Bootstrap у scss/_variables.scss. Деякі змінні мають значення null, ці змінні не виводять властивість, якщо вони не перевизначені у конфігурації.
Перевизначення змінних має відбуватися після імпорту функцій, але до решти імпорту.
Ось приклад, який змінює background-color та color для при імпорті та компіляції Bootstrap через npm:


    // Обов'язкові
@import "../node_modules/bootstrap/scss/functions";

// Перевизначення змінних за замовчуванням
$body-bg: #000;
$body-color: #111;

// Обов'язкові
@import "../node_modules/bootstrap/scss/variables";
@import "../node_modules/bootstrap/scss/variables-dark";
@import "../node_modules/bootstrap/scss/maps";
@import "../node_modules/bootstrap/scss/mixins";
@import "../node_modules/bootstrap/scss/root";

// Додаткові компоненти Bootstrap тут
@import "../node_modules/bootstrap/scss/reboot";
@import "../node_modules/bootstrap/scss/type";
// etc
