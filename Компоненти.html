КОМПОНЕНТИ

Кнопки

Використовуйте стилі кнопок Bootstrap, що налаштовуються, для дій у формах, діалогових вікнах і т.д. За допомогою кількох розмірів, станів і т.д.

Базовий клас:

Bootstrap має базовий клас .btn, який встановлює основні стилі, такі як відступи та вирівнювання вмісту. За замовчуванням елементи керування .btn мають прозору рамку та колір фону і не мають жодних явних стилів фокусу та наведення.


            <button type="button" class="btn">Базовий клас</button>

Клас .btn призначений для використання в поєднанні з варіантами кнопок або як основа для ваших власних стилів.

Варіанти

Bootstrap включає кілька варіантів кнопок, кожен з яких служить своїй семантичній меті, з додаванням декількох доповнень для більшого контролю.


    <button type="button" class="btn btn-primary">Primary</button>
    <button type="button" class="btn btn-secondary">Secondary</button>
    <button type="button" class="btn btn-success">Success</button>
    <button type="button" class="btn btn-danger">Danger</button>
    <button type="button" class="btn btn-warning">Warning</button>
    <button type="button" class="btn btn-info">Info</button>
    <button type="button" class="btn btn-light">Light</button>
    <button type="button" class="btn btn-dark">Dark</button>

    <button type="button" class="btn btn-link">Link</button>

Вимкнення перенесення тексту

Якщо Ви не бажаєте, щоб текст кнопки переносився, Ви можете додати до кнопки клас .text-nowrap. У Sass можна встановити $btn-white-space: nowrap, щоб вимкнути перенесення тексту для кожної кнопки.

Теги кнопок

Класи .btn призначені для використання з елементом <button>. Однак, Ви також можете використовувати ці класи для елементів <a> або <input> (хоча деякі браузери можуть застосовувати дещо інший рендеринг).
При використанні класів кнопок в елементах , які використовуються для запуску функціональності на сторінці (наприклад, згортання вмісту), замість посилатися на нові сторінки або розділи на поточній сторінці, цим посиланням слід присвоїти кнопку role="button", щоб належним чином донести свою мету до допоміжних технологій, таких як програми читання з екрану.


            <a class="btn btn-primary" href="#" role="button">Послання</a>
            <button class="btn btn-primary" type="submit">Кнопка</button>
            <input class="btn btn-primary" type="button" value="Поле введення">
            <input class="btn btn-primary" type="submit" value="Поле Відправити">
            <input class="btn btn-primary" type="reset" value="Поле Скинути">

Контурні кнопки

Потрібна кнопка, але не багаті кольори фону, які вони приносять? Замініть класи модифікаторів за замовчуваннямм на класи .btn-outline-*, щоб видалити всі фонові зображення та кольори на будь-якій кнопці.


<button type="button" class="btn btn-outline-primary">Primary</button>
<button type="button" class="btn btn-outline-secondary">Secondary</button>
<button type="button" class="btn btn-outline-success">Success</button>
<button type="button" class="btn btn-outline-danger">Danger</button>
<button type="button" class="btn btn-outline-warning">Warning</button>
<button type="button" class="btn btn-outline-info">Info</button>
<button type="button" class="btn btn-outline-light">Light</button>
<button type="button" class="btn btn-outline-dark">Dark</button>

Розміри

Бажаєте кнопки більшого чи меншого розміру? Додайте .btn-lg або .btn-sm додаткових розмірів.


    <button type="button" class="btn btn-primary btn-lg">Велика кнопка</button>
    <button type="button" class="btn btn-secondary btn-lg">Велика кнопка</button>
    <button type="button" class="btn btn-primary btn-sm">Маленька кнопка</button>
    <button type="button" class="btn btn-secondary btn-sm">Маленька кнопка</button>

Ви можете змінити свій розмір за допомогою змінних CSS:


    <button type="button" class="btn btn-primary"
    style="--bs-btn-padding-y: .25rem; --bs-btn-padding-x: .5rem; --bs-btn-font-size: .75rem;">
Користувацька кнопка
</button>

Відключений стан

Щоб кнопки були неактивними, додайте логічний атрибут disabled до будь-якого елемента <button>. До відключених кнопок застосовуються pointer-events: none, що запобігає спрацьовуванню станів наведення та активного стану.


<button type="button" class="btn btn-primary" disabled>Основна кнопка</button>
<button type="button" class="btn btn-secondary" disabled>Кнопка</button>
<button type="button" class="btn btn-outline-primary" disabled>Основна кнопка</button>
<button type="button" class="btn btn-outline-secondary" disabled>Кнопка</button>

    Відключені кнопки, що використовують елемент <a>, поводяться трохи інакше:

  • <a> не підтримують атрибут disabled, тому ви повинні додати клас .disabled, щоб він візуально виглядав вимкненим.
  • Включено деякі зручні для майбутнього стилі для відключення всіх pointer-events на кнопках прив'язки.
  • Вимкнені кнопки <a> повинні включати атрибут aria-disabled="true", щоб вказати стан елемента для допоміжних технологій.
  • Вимкнені кнопки <a> не повинні включати атрибут href.

    <a class="btn btn-primary disabled" role="button" aria-disabled="true">Первинне посилання</a>
    <a class="btn btn-secondary disabled" role="button" aria-disabled="true">Посилання</a>

Картки

Огляд

Карта - це гнучкий, розширюємий контейнер із вмістом. Він включає варіанти верхніх і нижніх колонтитулів, широкий вибір контенту, контекстні кольори фону і потужні параметри відображення. Якщо ви знайомі з Bootstrap 3, картки замінюють старі панелі, лунки та ескізи. Функціональність, аналогічна цим компонентам, доступна як класів модифікаторів для карток.

Приклад

Картки створюються з мінімальною кількістю розмітки та стилів, але все ж таки забезпечують тонну контролю та налаштування. Створені за допомогою flexbox, вони легко вирівнюються та добре поєднуються з іншими компонентами Bootstrap. За замовчуванням вони не мають margin, тому використовуйте утиліти відстані за необхідності.
Нижче наведено приклад базової картки зі змішаним змістом та фіксованою шириною. У карток немає фіксованої ширини для початку, тому вони природно заповнюють всю ширину батьківського елемента. Це легко налаштувати за допомогою різних параметрів розміру.


        <div class="card" style="width: 18rem;">
        <img src="..." class="card-img-top" alt="...">
        <div class="card-body">
          <h5 class="card-title">Заголовок картки</h5>
          <p class="card-text">Невеликий приклад тексту, який має ґрунтуватися на назві картки та складати основну частину вмісту картки.</p>
          <a href="#" class="btn btn-primary">Перейти куди-небудь</a>
        </div>
      </div>

Типи вмісту

Картки підтримують широкий спектр вмісту, включаючи зображення, текст, групи списків, посилання та багато іншого. Нижче наведено приклади того, що підтримується.

Тіло

Будівельним блоком картки є .card-body. Використовуйте його, коли Вам потрібен розділ на картці з м'якою підкладкою.


<div class="card">
  <div class="card-body">
    Це деякий текст в тілі картки.
  </div>
</div>

Заголовки, текст та посилання

Заголовки карток використовуються шляхом додавання .card-title до тега <h*>. Так само посилання додаються і розміщуються поруч один з одним шляхом додавання .card-link до тега <a>.
Підзаголовок використовується шляхом додавання .card-subtitle до тега <h*>. Якщо елементи .card-title та .card-subtitle розміщені в елементі .card-body, заголовок картки та підзаголовок будуть вирівняні.


        <div class="card" style="width: 18rem;">
        <div class="card-body">
          <h5 class="card-title">Заголовок картки</h5>
          <h6 class="card-subtitle mb-2 text-body-secondary">Підзаголовок картки</h6>
          <p class="card-text">Невеликий приклад тексту, який повинен ґрунтуватися на назві карти та складати основну частину вмісту картки.</p>
          <a href="#" class="card-link">Посилання картки</a>
          <a href="#" class="card-link">Друге посилання</a>
        </div>
      </div>

Зображення

.card-img-top та .card-img-bottom відповідно закруглюють верхній та нижній кути, щоб вони відповідали межам картки. За допомогою .card-text на карту можна додати текст. Текст усередині .card-text також можна стилізувати за допомогою стандартних HTML-тегів.


    <div class="card" style="width: 18rem;">
    <img src="..." class="card-img-top" alt="...">
    <div class="card-body">
      <p class="card-text">Невеликий приклад тексту, який має ґрунтуватися на назві картки та складати основну частину вмісту картки.</p>
    </div>
  </div>

Верхній та нижній колонтитули

Додайте до картки необов'язковий верхній та/або нижній колонтитули.


    <div class="card">
  <div class="card-header">
    Руокмендовані
  </div>
  <div class="card-body">
    <h5 class="card-title">Особливе звернення з заголовком</h5>
    <p class="card-text">З допоміжним текстом нижче як природний перехід до додаткового контенту.</p>
    <a href="#" class="btn btn-primary">Перейти куди-небудь</a>
  </div>
</div>

Карусель

Компонент слайд-шоу для циклічного переходу за елементами – зображеннями або текстовими слайдами – наприклад, каруселі.

Як це влаштовано

  • Карусель – це слайд-шоу для циклічного перегляду серії контенту, створеного за допомогою 3D-перетворень CSS та невеликої кількості JavaScript. Він працює з серією зображень, тексту або розмітки користувача. Він також включає підтримку попередніх/наступних елементів керування та індикаторів.
  • З міркувань продуктивності каруселі необхідно вручну ініціалізувати за допомогою методу конструктора каруселі. Без ініціалізації деякі прослуховувачі подій (зокрема, події, які потребують підтримки торкання/перегортання) не будуть зареєстровані доти, доки користувач явно не активує елемент керування або індикатор.
    Єдиним винятком є ​​автовідтворювані каруселі з атрибутом data-bs-ride="carousel", оскільки вони автоматично ініціалізуються під час завантаження сторінки. Якщо ви використовуєте каруселі з автоматичним відтворенням з атрибутом даних, не ініціалізуйте ті самі каруселі явно за допомогою методу конструктора.
  • Вкладені каруселі не підтримуються. Ви також повинні знати, що каруселі в цілому часто можуть викликати проблеми зі зручністю використання та доступністю.

Основні приклади

Ось базовий приклад каруселі із трьома слайдами. Зверніть увагу на попередній/наступні елементи керування. Рекомендовано використовувати елементи <button>, але ви також можете використовувати елементи <a> з role="button".


    <div id="carouselExample" class="carousel slide">
    <div class="carousel-inner">
      <div class="carousel-item active">
        <img src="..." class="d-block w-100" alt="...">
      </div>
      <div class="carousel-item">
        <img src="..." class="d-block w-100" alt="...">
      </div>
      <div class="carousel-item">
        <img src="..." class="d-block w-100" alt="...">
      </div>
    </div>
    <button class="carousel-control-prev" type="button" data-bs-target="#carouselExample" data-bs-slide="prev">
      <span class="carousel-control-prev-icon" aria-hidden="true"></span>
      <span class="visually-hidden">Попередній</span>
    </button>
    <button class="carousel-control-next" type="button" data-bs-target="#carouselExample" data-bs-slide="next">
      <span class="carousel-control-next-icon" aria-hidden="true"></span>
      <span class="visually-hidden">Наступний</span>
    </button>
  </div>

Каруселі не нормалізують автоматично розміри слайдів. Таким чином, вам може знадобитися використовувати додаткові утиліти або користувацькі стилі для відповідного розміру вмісту. Хоча каруселі підтримують елементи керування та індикатори «попередній/наступний», вони явно не потрібні. Додайте та налаштовуйте на свій розсуд
Ви повинні додати клас .active до одного зі слайдів, інакше карусель не буде видно. Також не забудьте встановити унікальний ID у файлі .carousel для додаткових елементів керування, особливо якщо ви використовуєте кілька каруселів на одній сторінці. Елементи керування та індикатори повинні мати атрибут data-bs-target (або href для посилань), який відповідає id елементу .carousel.

Індикатори

Ви можете додати індикатори в карусель разом із попередніми/наступними елементами керування. Індикатори дозволяють користувачам переходити безпосередньо до певного слайду.


<div id="carouselExampleIndicators" class="carousel slide">
  <div class="carousel-indicators">
    <button type="button" data-bs-target="#carouselExampleIndicators" data-bs-slide-to="0" class="active" aria-current="true" aria-label="Slide 1"></button>
    <button type="button" data-bs-target="#carouselExampleIndicators" data-bs-slide-to="1" aria-label="Slide 2"></button>
    <button type="button" data-bs-target="#carouselExampleIndicators" data-bs-slide-to="2" aria-label="Slide 3"></button>
  </div>
  <div class="carousel-inner">
    <div class="carousel-item active">
      <img src="..." class="d-block w-100" alt="...">
    </div>
    <div class="carousel-item">
      <img src="..." class="d-block w-100" alt="...">
    </div>
    <div class="carousel-item">
      <img src="..." class="d-block w-100" alt="...">
    </div>
  </div>
  <button class="carousel-control-prev" type="button" data-bs-target="#carouselExampleIndicators" data-bs-slide="prev">
    <span class="carousel-control-prev-icon" aria-hidden="true"></span>
    <span class="visually-hidden">Попередній</span>
  </button>
  <button class="carousel-control-next" type="button" data-bs-target="#carouselExampleIndicators" data-bs-slide="next">

  </button>
</div>

Підписи

Ви можете додавати підписи до своїх слайдів за допомогою елемента .carousel-caption у будь-якому .carousel-item. Їх можна легко приховати на невеликих вікнах перегляду, як показано нижче, за допомогою необов'язкових утиліт відображення. Спочатку ми приховуємо їх за допомогою .d-none, а повертаємо на пристрої середнього розміру за допомогою .d-md-block.

    <button type="button" data-bs-target="#carouselExampleCaptions" data-bs-slide-to="2" aria-label="Slide 3"></button>
  </div>
  <div class="carousel-inner">
    <div class="carousel-item active">
      <img src="..." class="d-block w-100" alt="...">
      <div class="carousel-caption d-none d-md-block">

Плавний перехід

Додайте .carousel-fade у свою карусель, щоб анімувати слайди з плавним переходом замість слайда. Залежно від вмісту вашої каруселі (наприклад, слайди, що містять тільки текст), ви можете додати .bg-body або будь-який користувальницький CSS до .carousel-item для правильного плавного переходу.


    <div id="carouselExampleFade" class="carousel slide carousel-fade">
  <div class="carousel-inner">
    <div class="carousel-item active">
      <img src="..." class="d-block w-100" alt="...">
    </div>

Ось що вам потрібно знати, перш ніж розпочати роботу з навігаційною панеллю:

  • Навігаційні панелі вимагають обгортки .navbar з .navbar-expand{-sm|-md|-lg|-xl|-xxl} для адаптивного згортання та класів колірної схеми.
  • Панелі навігації та їх вміст за промовчанням мінливі. Змініть контейнер, щоб обмежити їх ширину по горизонталі.
  • Використовуйте класи утиліт spacing та flex для керування інтервалами та вирівнюванням у панелях навігації.
  • Панелі навігації за промовчанням реагують, але ви можете легко змінити їх, щоб змінити це. Чуйна поведінка залежить від модуля, що підключається Collapse JavaScript.
  • Забезпечте доступність за допомогою елемента <nav> або, при використанні більш загального елемента, такого як <div>, додайте role="navigation" на кожну навігаційну панель, щоб явно ідентифікувати її як область-орієнтир для користувачів допоміжних технологій.
  • Вкажіть поточний елемент за допомогою aria-current="page" для поточної сторінки або aria-current="true" для поточного елемента в наборі.
  • Нове у версії 5.2.0: Панелі навігації можуть бути оформлені за допомогою змінних CSS, які прив'язані до базового класу .navbar. .navbar-light застарів, а .navbar-dark був переписаний для перевизначення змінних CSS замість додавання додаткових стилів.

Підтримуваний контент

  • .navbar-brand для Вашої компанії, продукту чи назви проекту.
  • .navbar-nav для повнорозмірної та полегшеної навігації (включаючи підтримку списків, що розкриваються).
  • .navbar-toggler для використання з плагіном згортання та іншими поведінками перемикання навігації.
  • Утиліти Flex та Spacing для будь-яких елементів керування та дій форми.
  • .navbar-textдля додавання вертикально центрованих рядків тексту.
  • .collapse.navbar-collapse для угруповання та приховування вмісту навігаційної панелі по батьківській контрольній точці.
  • Додайте необов'язковий .navbar-scroll, щоб встановити max-height і прокручувати розширений вміст навігаційної панелі.

Ось приклад усіх підкомпонентів, включених у адаптивну світлу навігаційну панель, яка автоматично згортається в контрольній точці lg (велика).


<nav class="navbar navbar-expand-lg bg-body-tertiary">
  <div class="container-fluid">
    <a class="navbar-brand" href="#">Навбар</a>
    <button class="navbar-toggler" type="button" data-bs-toggle="collapse" data-bs-target="#navbarSupportedContent" aria-controls="navbarSupportedContent" aria-expanded="false" aria-label="Перемикач навигації">
      <span class="navbar-toggler-icon"></span>
    </button>
    <div class="collapse navbar-collapse" id="navbarSupportedContent">
      <ul class="navbar-nav me-auto mb-2 mb-lg-0">
        <li class="nav-item">
          <a class="nav-link active" aria-current="page" href="#">Головна</a>
        </li>
        <li class="nav-item">
          <a class="nav-link" href="#">Посилання</a>
        </li>
        <li class="nav-item dropdown">
          <a class="nav-link dropdown-toggle" href="#" role="button" data-bs-toggle="dropdown" aria-expanded="false">
            Выпадаючий список
          </a>
          <ul class="dropdown-menu">
            <li><a class="dropdown-item" href="#">Дія</a></li>
            <li><a class="dropdown-item" href="#">Інша дія</a></li>
            <li><hr class="dropdown-divider"></li>

Бренд

.navbar-brand може застосовуватися до більшості елементів, але якір працює найкраще, оскільки для деяких елементів можуть знадобитися службові класи або стилі користувача.

Текст

Додайте текст до елемента за допомогою класу .navbar-brand.


<!-- As a link -->
<nav class="navbar bg-body-tertiary">
  <div class="container-fluid">
    <a class="navbar-brand" href="#">Панель навигації</a>
  </div>
</nav>

<!-- As a heading -->
<nav class="navbar bg-body-tertiary">
  <div class="container-fluid">
    <span class="navbar-brand mb-0 h1">Панель навигації</span>
  </div>
</nav>

Зображення

Ви можете замінити текст усередині .navbar-brand на <img>.

Зображення та текст

Ви також можете використовувати деякі додаткові утиліти для одночасного додавання зображення та тексту. Зверніть увагу на додавання .d-inline-block і .align-text-top до <img>.


  <nav class="navbar bg-body-tertiary">
  <div class="container-fluid">
    <a class="navbar-brand" href="#">
      <img src="/docs/5.3/assets/brand/bootstrap-logo.svg" alt="Logo" width="30" height="24" class="d-inline-block align-text-top">
      Bootstrap
    </a>
  </div>
</nav>

Навігація

Навігаційні посилання на панелі навігації засновані на параметрах .nav з їх власним класом-модифікатором і вимагають використання перемикач класу для правильного адаптивного стилю. Навігація в навігаційних панелях також збільшуватиметься, щоб займати якнайбільше горизонтального простору, щоб вміст Вашої навігаційної панелі був надійно вирівняний.
Додайте клас .active до .nav-link, щоб вказати поточну сторінку.
Зверніть увагу, що Вам також слід додати атрибут aria-current до активного .nav-link.


  <nav class="navbar navbar-expand-lg bg-body-tertiary">

Стан завантаження

Вкажіть стан завантаження компонента або сторінки за допомогою дзиги Bootstrap, повністю побудованих з використанням HTML, CSS і без JavaScript.

Огляд

Bootstrap «спінери» можна використовувати для відображення стану завантаження у проектах. Вони побудовані лише за допомогою HTML та CSS, а це означає, що для їх створення не потрібний спеціальній JavaScript. Однак Вам знадобиться спеціальний JavaScript, щоб переключити їхню видимість. Їхній зовнішній вигляд, розташування та розмір можна легко налаштувати за допомогою чудових службових класів.
Для зручності тут кожен завантажувач включає role="status" і вкладений <span class="visually-hidden">;Завантаження...</span>.

Спіннер із кордоном

Використовуйте спінери з кордоном для легкого індикатора завантаження.


  <div class="spinner-border" role="status">
  <span class="visually-hidden">Завантаження...</span>
</div>

Кольори

Спіннер з кордоном використовує currentColor як свій border-color, що означає, що Ви можете налаштувати колір за допомогою утиліт кольору тексту. Ви можете використовувати будь-яку з наших утиліт кольору тексту на стандартному спіннері.

</div>

Спіннер, що збільшується

Якщо Вам не подобається спіннер з межами, перейдіть на спіннер, що збільшується. Хоча технічно він не обертається, він постійно зростає!


  <div class="spinner-grow" role="status">
  <span class="visually-hidden">Завантаження...</span>
</div>

Знову ж таки, цей спіннер побудований за допомогою currentColor, так що ви можете легко змінити його зовнішній вигляд за допомогою утиліт кольору тексту. Тут він виділено синім кольором разом із підтримуваними варіантами.

Вирівнювання

Спінери в Bootstrap побудовані за допомогою rem, currentColor і display: inline-flex. Це означає, що їх можна легко змінити розмір, перефарбувати та швидко вирівняти.
